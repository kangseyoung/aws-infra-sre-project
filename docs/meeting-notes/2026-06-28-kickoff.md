# 2026-06-28 Kickoff Meeting

## 1. 프로젝트 목표

초보자 4명이 2주 동안 AWS 인프라 프로젝트를 진행합니다.

목표는 단순히 웹페이지를 띄우는 것이 아니라, AWS 위에서 네트워크, 서버, 보안, 모니터링, 장애 대응 흐름을 직접 구축하고 이해하는 것입니다.

## 2. 목표 아키텍처

```text
사용자 -> ALB -> EC2 -> Docker / Nginx -> CloudWatch
```

각 구성 요소가 어떤 역할을 하는지, 요청이 어떤 흐름으로 이동하는지 설명할 수 있게 되는 것을 목표로 합니다.

## 3. 역할 배정

| Member | Role |
| --- | --- |
| 강세영 | Compute + Network |
| 권태욱 | Observability / Platform / IaC 보조 / Compute 보조 |
| 김태우 | Terraform / IaC |
| 박찬혁 | Security + SRE 보조 |

## 4. 다음 회의 전까지 할 일

- 각자 담당 파트의 Must / Optional 범위를 정리합니다.
- 정리 위치: `docs/must-optional/`
- 템플릿: `templates/must-optional-template.md`
- 조사용 GPT 프롬프트: `templates/part-research-prompt.md`

## 5. 매일 공부기록 업로드

- 각자 매일 공부기록을 GitHub에 업로드합니다.
- 정리 위치: `docs/study-log/{name}/YYYY-MM-DD.md`
- 템플릿: `templates/study-log-template.md`

공부기록은 완벽한 정리본이 아니어도 됩니다.

우선순위는 다음과 같습니다.

1. 오늘 무엇을 봤는지
2. 무엇을 이해했는지
3. 무엇이 아직 헷갈리는지
4. GPT에게 무엇을 물어봤는지
5. 내일 무엇을 할지

## 6. 다음 회의 공유 방식

다음 회의에서는 각자 담당 파트 조사 내용을 2~3분 정도로 공유합니다.

공유할 내용:

- 내 파트가 전체 아키텍처에서 하는 역할
- 초보자가 꼭 알아야 할 핵심 개념
- Must / Optional 후보
- 다른 파트와 연결되는 부분
- GitHub에 남길 수 있는 산출물

## 7. Team Principle

- 완벽함보다 이해와 기록을 우선합니다.
- 모르는 내용을 숨기지 않고 기록합니다.
- 각자 맡은 파트를 설명할 수 있는 수준까지 정리합니다.
- 너무 넓게 잡지 않고 2주 안에 가능한 범위로 나눕니다.

## Notes

- AWS Access Key, Secret Key, `.pem`, `.env`, `terraform.tfstate`, `.terraform/`은 절대 커밋하지 않습니다.
- 비용이 발생할 수 있는 리소스는 실습 후 반드시 삭제합니다.
- 공부기록은 복붙보다 본인이 이해한 말로 정리하는 것을 우선합니다.
