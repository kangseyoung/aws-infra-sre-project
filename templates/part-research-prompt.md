# 담당 파트 조사용 GPT 프롬프트

나는 초보자 4명이 2주 동안 진행하는 AWS 인프라 프로젝트에 참여하고 있다.

목표 아키텍처는 다음과 같다.

사용자 → ALB → EC2 → Docker / Nginx → CloudWatch

프로젝트 목표는 단순히 웹페이지를 띄우는 것이 아니라, AWS 위에서 네트워크, 서버, 보안, 모니터링, 장애 대응 흐름을 직접 구축하고 이해하는 것이다.

내 담당 파트는 [여기에 본인 파트 입력]이다.

예시:

- Compute + Network
- Observability / Platform
- Terraform / IaC
- Security + SRE

내 담당 파트 기준으로 아래 내용을 정리해줘.

1. 이 파트가 전체 아키텍처에서 어떤 역할을 하는지
2. 초보자가 꼭 알아야 할 핵심 개념
3. 2주 프로젝트에서 반드시 해야 할 Must 범위
4. 시간이 남으면 할 Optional 범위
5. 다른 파트와 연결되는 부분
6. GitHub에 남기면 좋은 산출물
7. 다음 회의에서 2~3분 안에 공유할 수 있는 요약

조건:

- 초보자 기준으로 설명해줘.
- 너무 넓게 잡지 말고 2주 안에 가능한 범위로 나눠줘.
- Must와 Optional을 명확히 구분해줘.
- 포트폴리오에 남길 수 있는 결과물 중심으로 정리해줘.
